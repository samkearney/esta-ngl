Hierarchy of Interfaces¶

Interfaces can be instantiated in a hierarchy. The interface instance hierarchy present in a device definition provides meaningful information about how features are organized in a device.

For example, consider the definition for a lighting fixture with a few individually-addressable cells. The fixture may organize its cells into cell groups, each containing a master intensity that controls all of the cells:

<interfacedef class="cell-group">
  <property class="org.esta.intensity.1/intensity" id="intensity-master" />
  <child class="cell" minimum="1" />
</interfacedef>

Note the Child element here. This element’s job is to set rules for what children are allowed when the interface is instantiated. This particular one says that a cell-group interface instance must have at least one cell child.

The cell interface might look like:

<interfacedef class="cell">
  <property class="org.esta.intensity.1/intensity" id="intensity" />
  <property class="org.esta.color.1/rgb" id="color" />
</interfacedef>

The cell has individual intensity and color properties.

These interface definitions give this device the tools to communicate information about its cell layout. Assuming the definitions above appear in a definition list with the class “com.acme.definitions.1”, the device might contain:

<!-- A group of 4 cells -->
<interface class="com.acme.definitions.1/cell-group" id="cell-group-1">
  <interface class="com.acme.definitions.1/cell" id="1" />
  <interface class="com.acme.definitions.1/cell" id="2" />
  <interface class="com.acme.definitions.1/cell" id="3" />
  <interface class="com.acme.definitions.1/cell" id="4" />
</interface>

To address the intensity property of cell 2, you would use the qualified ID cell-group-1/2/intensity.

Child¶

Interface definitions can contain child objects which set rules for which interface instances are allowed as children of that interface.

Attributes¶

Class¶

The class of an interface that is allowed as a child to this interface. This can be a Qualified Class or a relative class.

Minimum¶

The minimum number of instances of the child interface that are allowed. If this attribute is absent, the minimum is assumed to be 0.

Maximum¶

The maximum number of instances of the child interface that are allowed. If this attribute is absent, the maximum is assumed to be infinite.

Exclusive Group¶

The name of an exclusive group that this child is a member of, if instantiated. See Exclusive Group.

Markup¶

XML

  • Tag name: child

  • Attributes:

    • class: Class

    • minimum: Minimum

    • maximum: Maximum

    • exclusivegroup: Exclusive Group

Example:

<child class="com.acme.definitions.1/cell" minimum="1" maximum="10" />
